def forward(X, W1, b1, W2, b2):
    Z = 1 / (1 + np.exp(-X.dot(W1) - b1))
    A = Z.dot(W2) + b2
    expA = np.exp(A)
    Y = expA / expA.sum(axis=1, keepdims=True)
    return Y, Z


# determine the classification rate
# num correct / num total
def classification_rate(Y, P):
    n_correct = 0
    n_total = 0
    for i in range(len(Y)):
        n_total += 1
        if Y[i] == P[i]:
            n_correct += 1
    return float(n_correct) / n_total


def derivative_w2(Z, T, Y):
    N, K = T.shape
    M = Z.shape[1] # H is (N, M)

    ret4 = Z.T.dot(T - Y)

    return ret4


def derivative_w1(X, Z, T, Y, W2):
    N, D = X.shape
    M, K = W2.shape

    dZ = (T - Y).dot(W2.T) * Z * (1 - Z)
    ret2 = X.T.dot(dZ)

    # assert(np.abs(ret1 - ret2).sum() < 0.00001)

    return ret2


def derivative_b2(T, Y):
    return (T - Y).sum(axis=0)


def derivative_b1(T, Y, W2, Z):
    return ((T - Y).dot(W2.T) * Z * (1 - Z)).sum(axis=0)


def cost(T, Y):
    tot = T * np.log(Y)
    return tot.sum()


def train(X, Y):
    # create the data
    D = X.shape[1] # dimensionality of input
    M = 10 # hidden layer size
    K = len(set(Y)) # number of classes


    N = len(Y)
    # turn Y into an indicator matrix for training
    T = np.zeros((N, K))
    for i in range(N):
        T[i, Y[i]] = 1

 

    # randomly initialize weights
    W1 = np.random.randn(D, M)
    b1 = np.random.randn(M)
    W2 = np.random.randn(M, K)
    b2 = np.random.randn(K)

    learning_rate = 1e-3
    costs = []
    for epoch in range(10000):
        output, hidden = forward(X, W1, b1, W2, b2)
        if epoch % 100 == 0:
            c = cost(T, output)
            P = np.argmax(output, axis=1)
            r = classification_rate(Y, P)
            print("cost:", c, "classification_rate:", r)
            costs.append(c)

        # this is gradient ASCENT, not DESCENT
        # be comfortable with both!
        # oldW2 = W2.copy()

        gW2 = derivative_w2(hidden, T, output)
        gb2 = derivative_b2(T, output)
        gW1 = derivative_w1(X, hidden, T, output, W2)
        gb1 = derivative_b1(T, output, W2, hidden)

        W2 += learning_rate * gW2
        b2 += learning_rate * gb2
        W1 += learning_rate * gW1
        b1 += learning_rate * gb1
        
    return W1, b1, W2, b2

        
def predict(X, W1, b1, W2, b2):
    Y, _ = forward(X, W1, b1, W2, b2)
    predictions = np.argmax(Y, axis=1)
    return predictions

Hidden Layers and Network Performance :

Exploring the impact of hidden layer neuron counts on neural network performance reveals a delicate balancing act. Initially, as we augment the number of neurons, the network's accuracy ascends. This boost aligns with expectations; heightened complexity empowers the model to discern subtler data patterns. With more neurons at its disposal, the network becomes adept at capturing intricate relationships between features and desired outputs.

However, a tipping point emerges. Continuing to augment neuron counts leads to a decline in accuracy. This phenomenon stems from overfitting, where the model's excessive flexibility prompts it to memorize training data rather than generalize rules. Essentially, it's akin to cramming for a one-time exam.

Identifying the sweet spotthe optimal neuron countrests on several factors. Data complexity plays a pivotal role; intricate datasets may necessitate a higher neuron count to unravel their mysteries. The size of the training set also influences this balance; larger datasets enable effective handling of more parameters, potentially pushing the optimal neuron count higher. Moreover, network architecture matters; deeper networks often require fewer neurons per layer compared to shallower ones to grasp complex patterns.

Determining the ideal neuron count entails a blend of art and science, involving experimentation with various configurations. Techniques like validation sets prove invaluable in detecting overfitting and steering the model towards optimal performance.

In essence, this exploration underscores the quest for equilibrium. Too few neurons lead to underfitting, overlooking critical patterns, while an excess triggers overfitting, excelling on training data but floundering on unseen examples. The objective is to craft a network robust enough to learn without losing sight of generalization principles.

5.Compare the performance of the neural network models with logistic regression, discussing the strengths and weaknesses of each approach.

Logistic Regression:

Strengths: Simplicity: Logistic regression is a simple and interpretable algorithm, making it easy to understand and implement. Efficiency: It's computationally efficient, especially for small datasets with linearly separable classes. Interpretability: Logistic regression provides coefficients for each feature, allowing interpretation of the impact of each feature on the prediction.

Weaknesses: Limited Complexity: Logistic regression assumes a linear relationship between features and the log-odds of the target variable. It might struggle to capture complex non-linear relationships in the data. Feature Engineering: Logistic regression heavily relies on feature engineering to represent non-linear relationships or interactions between features effectively. High Bias: Logistic regression might underfit complex datasets if the relationship between features and the target variable is non-linear.

Neural Network Models:

Strengths: Flexibility: Neural networks can capture complex non-linear relationships in the data due to their hierarchical structure and non-linear activation functions. Feature Learning: Neural networks can automatically learn useful features from raw data, reducing the need for extensive feature engineering. Scalability: Neural networks can scale to handle large datasets and high-dimensional feature spaces effectively.

Weaknesses: Complexity: Neural networks are complex models with many parameters, making them harder to interpret and prone to overfitting, especially on small datasets. Computational Intensity: Training neural networks can be computationally intensive, especially for deep architectures or large datasets, requiring substantial computational resources. Hyperparameter Tuning: Neural networks require careful tuning of hyperparameters such as learning rate, number of layers, neurons per layer, etc., which can be time-consuming and require expertise.

Comparison:

Performance: Neural networks generally outperform logistic regression on complex tasks with large datasets and non-linear relationships.

Interpretability: Logistic regression provides interpretable results with clear coefficients, making it suitable for scenarios where interpretability is crucial. Neural networks lack interpretability due to their complex nature.

Computational Resources: Logistic regression is computationally efficient and requires fewer computational resources compared to neural networks, making it suitable for low-resource environments or real-time applications.

Data Size: Logistic regression performs well on small to medium-sized datasets with linearly separable classes. Neural networks excel when dealing with large datasets or when complex patterns need to be learned.

In summary, the choice between logistic regression and neural networks depends on the specific characteristics of the dataset, the task at hand, computational resources, and the importance of interpretability. Logistic regression is suitable for simpler tasks with linear relationships, interpretability requirements, or limited computational resources. On the other hand, neural networks are preferred for complex tasks with non-linear relationships, large datasets, and when interpretability is not the primary concern.
